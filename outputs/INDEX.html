👁️ Visão Geral do Sistema

Sistema completo de análise de iluminação noturna urbana utilizando imagens de satélite TIFF.

Características Principais

✅ Análise de 1200+ imagens de satélite
✅ Processamento paralelo de 8-10x mais rápido
✅ Cobertura de 60+ cidades
✅ Histórico de 10 anos (2014-2024)
✅ Interface web moderna e responsiva
✅ Detecção automática de crescimento de luz
✅ Geração de gráficos e tendências
✅ API REST completa
✅ Sistema de cache inteligente
✅ Automação por agendador de tarefas
✅ Sugestão automática de thresholds

🏗️ Arquitetura do Sistema

O sistema é dividido em 3 camadas principais:

1️⃣ Interface Web (Flask)

Aplicação Flask moderna com Bootstrap 5 e Leaflet.js para visualização geográfica.

2️⃣ Camada de Processamento

Módulos Python especializados em análise de imagens, processamento paralelo e tendências.

3️⃣ Armazenamento de Dados

Dados em formato CSV e cache JSON para performance otimizada.

🔌 APIs REST

O sistema oferece 8 endpoints REST para integração com outros sistemas:

Método Endpoint Descrição
GET /api/listar-imagens Lista todas as imagens disponíveis
POST /api/analisar Analisa arquivo TIFF selecionado
POST /api/comparar-crescimento Compara crescimento entre dois arquivos
GET /api/sugerir-thresholds Sugestão de thresholds
GET /api/pastas-cidades Lista cidades disponíveis

⚙️ Componentes Técnicos

O sistema é composto por 6 módulos Python principais:

1. leitura_de_luz.py

AnalisadorLuzSatelite - Análise de imagens TIFF

2. processador_paralelo.py

ProcessadorParalelo - Processamento multi-thread

3. tendencias.py

AnalisadorTendencias - Análise e gráficos

4. sugestores_threshold.py

SugestorThreshold - Sugestão inteligente de limiares

5. app.py

Flask App - Servidor web e API REST

6. main.py

Menu CLI - Interface de linha de comando

🔧 Troubleshooting

❌ "Nenhuma pasta encontrada"

✅ Verificar se as imagens estão em C:\Users\{usuario}\Documents\

❌ "Erro ao analisar TIFF"

✅ Verificar se arquivo é GeoTIFF válido com gdalinfo

❌ "Processamento muito lento"

✅ Aumentar limite de threads de 4 para 8

❌ "Porta 5000 já em uso"

✅ Executar em porta diferente: python app.py --port 5001
